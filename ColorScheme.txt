========================================================================
DOCUMENTO DE ESTRATÉGIA DE CORES E UI - MATERIAL DESIGN 3 (MD3)
========================================================================
Este documento é a ÚNICA FONTE DE VERDADE para a aplicação de cores e 
estilos na aplicação. A aplicação utiliza React Native Paper e suporta 
múltiplos temas dinâmicos (Light/Dark Mode e Paletas de Marca).

REGRAS GERAIS E ABSOLUTAS (PROIBIDO IGNORAR):
1. É ESTRITAMENTE PROIBIDO usar Hex Codes (ex: #FFFFFF, #000) no código dos ecrãs (exceto no ficheiro themes.js e CustomSnackBar).
2. É ESTRITAMENTE PROIBIDO usar classes de cor do Tailwind (ex: bg-white, text-black, border-gray-200, bg-red-500).
3. Todas as cores devem ser injetadas através do hook: `const theme = useTheme();`.
4. Todos os ecrãs devem suportar Dark Mode nativamente, o que significa que o contraste é garantido pelo emparelhamento correto das variáveis (ex: usar sempre `onPrimary` por cima de `primary`).

------------------------------------------------------------------------
1. FUNDOS, ECRÃS E CARTÕES (ELEVAÇÃO TONAL)
Não usamos sombras pesadas. A separação visual é feita por camadas de cor.

* Raiz do Ecrã (SafeAreaView / View principal): 
  - backgroundColor: theme.colors.background

* Cartões Principais (Surface de listas, negócios, campanhas): 
  - backgroundColor: theme.colors.surfaceContainer (ou secondaryContainer para destacar)
  - border: 1px solid theme.colors.outlineVariant (Nunca usar outline normal para cartões)
  - elevation: 1 a 2 (para o iOS processar a sombra suave)

* Blocos de Informação Internos / Formulários / Placeholders de Imagem:
  - backgroundColor: theme.colors.surfaceVariant

* Modais, Diálogos e Bottom Sheets:
  - backgroundColor: theme.colors.surfaceContainerHighest

------------------------------------------------------------------------
2. TIPOGRAFIA E ÍCONES (HIERARQUIA DE LEITURA)
A cor do texto deve SEMPRE corresponder ao fundo onde está assente.

* Títulos Principais do Ecrã (Headline): 
  - color: theme.colors.primary (Dá destaque à marca da página)

* Texto Normal sobre o 'background' da App:
  - color: theme.colors.onBackground

* Títulos dentro de Cartões (TitleLarge/Medium):
  - color: theme.colors.onSurface (ou onSecondaryContainer se o cartão for secondaryContainer)

* Corpo de Texto, Descrições e Ícones Secundários:
  - color: theme.colors.onSurfaceVariant (Cria o efeito editorial de leitura suave)

------------------------------------------------------------------------
3. BOTÕES E CHAMADAS DE AÇÃO (CTAs)
Os botões devem ter a forma de pílula (borderRadius: 9999) ou seguir a regra do tema (roundness).

* Ação Primária (Gravar, Iniciar Sessão, Aceitar):
  - backgroundColor: theme.colors.primary
  - color (Texto/Ícone): theme.colors.onPrimary

* Ação Secundária (Cancelar, Voltar, Filtrar):
  - backgroundColor: theme.colors.surfaceVariant (ou primaryContainer para dar mais ênfase)
  - color: theme.colors.onSurfaceVariant (ou onPrimaryContainer)

* Ação Destrutiva (Eliminar, Descartar, Rejeitar):
  - backgroundColor: theme.colors.errorContainer
  - color: theme.colors.onErrorContainer

------------------------------------------------------------------------
4. INTERATIVIDADE (TOUCHABLE RIPPLE)
Sempre que um elemento for clicável (cartões da lista, imagens de perfil):

* A Regra do Ripple: O `TouchableRipple` NUNCA deve ter `backgroundColor`. O fundo pertence à `View` ou `Surface` que o envolve.
* A Cor da Onda (Ripple Color): 
  - Se for um cartão claro/escuro normal: `rippleColor="rgba(150, 150, 150, 0.2)"`
  - Evitar usar hex + opacidade dinâmica (ex: `+ '33'`) para não quebrar no iOS/Android.

------------------------------------------------------------------------
5. INPUTS E FORMULÁRIOS (TextInput)
* Fundo do Input: theme.colors.surfaceVariant
* Texto inserido: theme.colors.onSurface
* Borda Inativa: theme.colors.outlineVariant
* Borda Ativa/Focada: theme.colors.primary
* Placeholder: theme.colors.onSurfaceVariant

------------------------------------------------------------------------
6. EXCEÇÕES À REGRA
* CustomSnackBar: Os alertas de sucesso e erro devem manter as cores universais estáticas de trânsito (Verde = #16A34A, Vermelho = #DC2626, Texto = #FFFFFF) para garantir o reconhecimento imediato em qualquer tema.
* Overlays Escuros (Fundo de ecrãs de Login/Registo com imagem): Usar `backgroundColor: theme.colors.background + 'B3'` (Fundo com 70% de opacidade).
